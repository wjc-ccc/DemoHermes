# 简单 Loop 实现 - 7.13 版本

> 目标：打通「终端输入 content → 组装成 Message → ContextBuilder 组装 → model 发送给 AI → AI 回复」这条最小链路，作为 `core/loop.py` 的参考实现。
>
> 本文为**参考学习用示例代码**，不直接修改现有源文件。

---

## 一、数据流

```
终端 input(text)
   │
   ▼
observe  : 组装成 Message(role=user, content_text=text) → 塞进 Session.messages
   │
   ▼
think    : ContextBuilder.building_context() 把 Session → list[dict] (OpenAI 格式)
   │
   ▼
model.chat(messages) → AI 回复 str
   │
   ▼
act      : 组装成 Message(role=assistant) → 塞进 Session.messages
   │
   ▼
persist  : （简化版先留空，后续接 memory / trajectory）
```

对应 `loop.py` 注释里的四段：`observe → think → act → persist`。

---

## 二、当前代码里影响跑通的问题

先把这些修掉，链路才能通：

| 文件 | 问题 | 说明 |
|------|------|------|
| `types.py` | `class Message:` 没继承 `BaseModel` | 注释写了"使用 pydantic 保证类型安全"，但裸类里 `Field(...)` 只是挂个 `FieldInfo` 对象当类属性，**不会校验也不会生成默认值**。需 `class Message(BaseModel):` |
| `types.py` | `choices=[...]` | pydantic v2 的 `Field` 没有 `choices`，用 `typing.Literal[...]` 代替 |
| `types.py` | `dict[str:Any]` | 非法语法（冒号是字典字面量用的）；类型应写 `dict[str, Any]` |
| `types.py` | `Session` 引用了 `ToolResult` | 未定义，先注释掉或补定义，否则类体报 `NameError` |
| `context_builder.py` | `from types import Session` | ① 覆盖了标准库 `types` ② 应是相对导入 `from .types import Session` |
| `context_builder.py` | `building_context` 里 `self._loading_system_prompt` 漏 `()`、`return` 没返回值 | 方法调用和返回值都没生效 |
| `model/deepseek.py` | `def DeepSeekModel(BaseModel):` | 应是 `class DeepSeekModel(BaseModel):` |
| `model/deepseek.py` | `chat` 没有 `return` | 上层拿不到回复 |
| `model/deepseek.py` | `from baseStructure import BaseModel` | 应相对导入 `from .baseStructure import BaseModel` |

## 三、参考代码

### 2. `core/context_builder.py`

```python
"""
ContextBuilder - 把 Session 组装成发给 LLM 的 list[dict] (参考示例)
"""
from .types import Session


class ContextBuilder:
    def __init__(self, session: Session, system_prompt: str = ""):
        self.session = session
        self.system_prompt = system_prompt or "你是一个简洁的 agent，简短回答用户问题。"

    def _loading_system_prompt(self) -> dict:
        return {"role": "system", "content": self.system_prompt}

    def building_context(self) -> list[dict]:
        """1.system  2.历史消息  -> OpenAI messages 格式"""
        msgs: list[dict] = [self._loading_system_prompt()]
        for m in self.session.messages:
            msgs.append({"role": m.role, "content": m.content_text})
        return msgs


class ContextSaver:
    """后续：把当前上下文长度/时间落库"""
    def saving(self, builder: ContextBuilder):
        pass
```

### 3. `core/loop.py`（核心）

```python
"""
AgentLoop - 核心对话循环 (参考示例)

observe -> think -> act -> persist
"""
from .types import Session, Message, ContentPart
from .context_builder import ContextBuilder


class AgentLoop:
    def __init__(self, model, session: Session = None, system_prompt: str = ""):
        self.model = model                       # BaseModel 子类实例
        self.session = session or Session()
        self.builder = ContextBuilder(self.session, system_prompt)

    # observe：终端文本 -> Message(user) -> 进 Session
    def observe(self, text: str) -> Message:
        user_msg = Message(
            role="user",
            content_text=text,
            content=[ContentPart(type="text", text=text)],
        )
        self.session.add_message(user_msg)
        return user_msg

    # think：组装上下文 -> 调 LLM
    def think(self) -> str:
        messages = self.builder.building_context()
        return self.model.chat(messages)

    # act：把回复落成 Message(assistant)
    def act(self, reply: str) -> Message:
        asst_msg = Message(
            role="assistant",
            content_text=reply,
            content=[ContentPart(type="text", text=reply)],
        )
        self.session.add_message(asst_msg)
        return asst_msg

    # persist：简化版留空，后续接 memory/trajectory
    def persist(self, user_msg: Message, asst_msg: Message):
        pass

    # 一轮完整步骤
    def step(self, text: str) -> str:
        user_msg = self.observe(text)
        reply = self.think()
        asst_msg = self.act(reply)
        self.persist(user_msg, asst_msg)
        return reply
```

### 4. `model/deepseek.py`（修正后）

```python
"""
DeepSeek Provider (参考示例)
"""
from openai import OpenAI
from .baseStructure import BaseModel
from ..Provider import DEEPSEEK_API_KEY, DEEPSEEK_BASE_URL


class DeepSeekModel(BaseModel):          # ← class 不是 def
    def __init__(self):
        super().__init__()
        self.client = OpenAI(api_key=DEEPSEEK_API_KEY, base_url=DEEPSEEK_BASE_URL)

    def chat(self, messages: list[dict]) -> str:   # ← 必须 return
        response = self.client.chat.completions.create(
            model="deepseek-v4-flash",
            messages=messages,
        )
        return response.choices[0].message.content
```

### 5. `gateway/cli.py`（终端入口）

```python
"""
CLI - 终端交互入口 (参考示例)
    python -m Agent.gateway.cli
"""
from ..model.deepseek import DeepSeekModel
from ..core.loop import AgentLoop


def main():
    model = DeepSeekModel()
    loop = AgentLoop(
        model=model,
        system_prompt="你是一个测试版 agent loop，用户问什么你简单回答即可。",
    )
    print("Agent 已启动，输入 exit 退出。")
    while True:
        text = input("you > ").strip()
        if not text:
            continue
        if text.lower() in {"exit", "quit"}:
            break
        reply = loop.step(text)         # observe -> think -> act -> persist
        print(f"ai  > {reply}")


if __name__ == "__main__":
    main()
```

## 五、后续扩展方向

主循环结构定下来后，往里加东西都不用动 `step` 的骨架：

- **工具调用**：在 `think` 和 `act` 之间插一层「解析 `tool_calls` → 执行 → 把结果回注成新 Message → 再 `think`」，循环直到没有 `tool_calls`。
- **记忆**：`persist` 里接 `memory/manager.py`，写 episodic + hot snapshot；`ContextBuilder` 里加 `_loading_hot_memory`。
- **Skill**：`ContextBuilder` 里加 `_loading_skills`，按当前问题渐进加载。
- **事件**：在 `observe/think/act/persist` 各节点发 `events`，供 gateway 推 WebSocket。
