## 推荐开发顺序（6 个阶段）

**原则：每完成一阶段就能独立自测，不依赖后面阶段。**

```
阶段0 地基 → 阶段1 能对话 → 阶段2 能调工具 → 阶段3 有记忆 → 阶段4 可观测 → 阶段5 Claude SDK → 阶段6 进化+Skill
```

---

### 阶段 0：地基加固（0.5 天）

**先写这些：**

| 顺序 | 文件 | 原因 |
|------|------|------|
| 1 | `core/types.py` | 后面所有模块都依赖 Message / Turn / ToolCall |
| 2 | `model/deepseek.py` 重构为 Provider 类 | 把测试脚本变成正式实现 |
| 3 | `model/registry.py` | 统一获取模型，方便后续切换 |
| 4 | `logging/loggingTool.py` | 尽早有日志，后面调试省力 |

**自测：**

```bash
# 1. 模型能返回
python -c "from agent.model.registry import get_model; print(get_model().chat([{'role':'user','content':'hi'}]))"

# 2. 日志能输出
python -m logging.loggingTool
```

**通过标准：** 模型有回复，日志按 `LOG_LEVEL` 正常打印。

---

### 阶段 1：最小对话闭环（1 天）— 最高优先级

**先写这些：**

| 顺序 | 文件 |
|------|------|
| 1 | `core/session.py` — 创建/维护会话 |
| 2 | `prompt/builder.py` + `templates/identity.md` — 最简 system prompt |
| 3 | `core/loop.py` — 仅 observe → think → persist（先不要工具） |
| 4 | `gateway/cli.py` — 终端入口 |

**自测：**

```bash
python -m agent.gateway.cli
# 输入：你好
# 期望：Agent 正常回复，多轮对话上下文不丢
```

**通过标准：**

- CLI 能连续对话 3 轮以上
- 同一 session 内能引用上一轮内容
- 异常（API 超时、空输入）有基本处理

> 这一阶段完成后，就有了一个「能聊天的 Agent」，后面所有能力都挂在这个 loop 上。

### 阶段 2：工具能力（1–1.5 天）

**先写这些：**

| 顺序 | 文件 |
|------|------|
| 1 | `tools/base.py` + `tools/registry.py` |
| 2 | `tools/file_tools.py` — 先做 `read_file` |
| 3 | `tools/bash_tools.py` — 先做 `run_command`（加超时） |
| 4 | 改 `core/loop.py` — 加入 act 阶段（tool call 循环） |
| 5 | `prompt/templates/tools_guide.md` — 填入工具说明 |

**自测：**

```bash
python -m agent.gateway.cli
```

测试用例：

| 输入 | 期望行为 |
|------|----------|
| `读取 README.md 并总结` | 调用 read_file，返回摘要 |
| `列出当前目录文件` | 调用 list_dir 或 bash |
| `执行 echo hello` | bash 返回 hello |

**通过标准：**

- LLM 能主动发起 tool_call
- 工具结果能回注上下文并生成最终回复
- 终端能看到「调用了什么工具、返回了什么」

## 不要做的事（避免绕路）

1. **不要先做 evolution** — 没有 loop 和 memory，进化无数据可处理
2. **不要三层 memory 同时写** — 先做 L1，再做 L2
3. **不要先做可视化前端** — 先把 `events.py` + API 跑通，前端最后接
4. **不要在各文件重复 load_dotenv** — 统一走 `Config.py`
5. **不要追求完美 tool schema** — 先 2 个工具跑通，再扩展

## 第一行代码写哪里

如果今天只动手一件事，按这个顺序：

```
core/types.py  →  model/deepseek Provider  →  core/loop.py  →  gateway/cli.py
```

这四个文件跑通后，就有了一个能对话的 Agent，后面所有模块都是往这个 loop 上挂插件。

## 模块依赖关系（简图）

```
用户输入 → gateway → core/loop → model（思考）
                              → tools（行动）
                              → memory（持久化）
                              → evolution（后台进化）
```

各阶段与模块对应：

| 阶段 | 涉及模块 |
|------|----------|
| 0 | `core/types`, `model/`, `logging/` |
| 1 | `core/loop`, `core/session`, `prompt/`, `gateway/cli` |
| 2 | `tools/`, `core/loop`（扩展） |
| 3 | `memory/hot`, `memory/episodic`, `memory/manager` |
| 4 | `core/events`, `gateway/api` |
| 5 | `model/claude_sdk` |
| 6 | `memory/procedural`, `evolution/` |
