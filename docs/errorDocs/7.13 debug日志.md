## 1. pydantic 类没继承 BaseModel

`class Message:` 裸类里写 `Field(...)` 只是把 FieldInfo 挂成类属性，不会校验、也不会生成默认值。
解决办法：`class Message(BaseModel):`，import 里加上 `BaseModel`。

## 2. Field 用了 choices 参数

pydantic v2 的 `Field` 没有 `choices`，传了会报未知参数。
解决办法：类型注解改用 `Literal[...]`，如 `role: Literal["user","assistant","system"]`；Literal 字段要给合法默认值 `default="user"`。

## 3. dict[str:Any] 语法错误

冒号是字典字面量的写法，类型下标必须用逗号；这个直接导致模块 import 失败。
解决办法：`dict[str, Any]`。

## 4. default_factory=str|None 不可调用

`str|None` 是 UnionType，无法 `()` 实例化，实例化模型时 TypeError（`dict|None` 同理）。
解决办法：Optional 字段直接 `default=None`（path / url / data / error / result_text 等）。

## 5. Session 引用未定义的 ToolResult

类体执行到 `tool_result:list[ToolResult]` 时 `ToolResult` 还没定义，NameError。
解决办法：已移除该字段（如后续需要，先在 Session 之前定义 `ToolResult(BaseModel)`）。

## 6. context_builder 导入覆盖标准库

`from types import Session` 既覆盖了标准库 `types`，又不是相对导入，作为包导入时找不到。
解决办法：`from .types import Session`，顺手删掉无用的 `from email import message`。

## 7. 方法调用漏 () / 方法漏 self / 没 return

`self._loading_system_prompt` 没加 `()` 等于没调用；`_loading_skills()` 漏 `self` 形参，用实例调用会 TypeError；`building_context` 没 return，上层拿到 None。
解决办法：补 `()`、补 `self`、最后 `return self.messages`；并补 `_loading_messages` 把 Session 里的消息真正拼进去。

## 8. system prompt 硬编码在 builder 里

prompt 内容写死在代码里，改 prompt 要动代码。
解决办法：prompt 放 `agent/prompt/systemPrompt.md`，路径 `SYSTEM_PROMPT_PATH` 统一放配置中心 `Provider.py`，builder 里 `SYSTEM_PROMPT_PATH.read_text(encoding="utf-8")` 加载。

## 9. deepseek 把 class 写成 def

`def DeepSeekModel(BaseModel):` 是函数定义不是类，无法继承。
解决办法：改成 `class DeepSeekModel(BaseModel):`。

## 10. 包内用了绝对导入

`from baseStructure import BaseModel` 在以包（`Agent.model.deepseek`）方式导入时找不到模块。
解决办法：相对导入 `from .baseStructure import BaseModel`，`from ..Provider import ...`。
